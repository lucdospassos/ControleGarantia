COMO GERAR O APK RELEASE ASSINADO PELO GITHUB ACTIONS

OBJETIVO
Gerar um APK release assinado para distribuir o aplicativo aos clientes.

IMPORTANTE
Nunca envie o arquivo .jks para o repositório GitHub.
Nunca coloque senhas dentro dos arquivos do projeto.
A chave release deve ser guardada com segurança, pois as futuras atualizações do app precisam ser assinadas com a mesma chave.

PASSO 1 - GERAR A CHAVE NO WINDOWS
1. Instale o Java JDK 17 ou superior, caso o comando keytool ainda não exista no Windows.
2. Entre na pasta scripts do projeto.
3. Execute:

   gerar_chave_release_windows.bat

4. O script vai criar:

   controlelacres-release-key.jks
   android-keystore-base64.txt

5. Guarde o arquivo controlelacres-release-key.jks em local seguro, fora do GitHub.

PASSO 2 - CRIAR OS SECRETS NO GITHUB
No repositório, abra:

Settings > Secrets and variables > Actions > New repository secret

Cadastre estes 4 secrets:

1) ANDROID_KEYSTORE_BASE64
   Cole todo o conteúdo do arquivo android-keystore-base64.txt.

2) RELEASE_STORE_PASSWORD
   Cole a senha principal da chave, digitada no script.

3) RELEASE_KEY_ALIAS
   Use:
   controlelacres

4) RELEASE_KEY_PASSWORD
   Cole a senha da chave, digitada no script.

PASSO 3 - RODAR O WORKFLOW
Depois de configurar os secrets:

Actions > Gerar APK Android > Run workflow

Quando terminar verde, baixe o artifact:

ControleLacresGarantia-v1.5-release-assinado-apk

Dentro dele estará o APK release assinado.

SE NÃO APARECER O RELEASE
Se aparecer apenas o artifact debug, significa que os secrets não foram configurados corretamente ou estão com nomes diferentes.

DICA IMPORTANTE
A primeira versão release assinada instalada no celular será a base para as próximas atualizações.
Para atualizar por cima no Android, as próximas versões precisam usar a mesma chave .jks.
